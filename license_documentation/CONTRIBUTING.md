Thanks for wanting to contribute! This document explains the process for code, docs, and issue work.

## Code of Conduct
Follow `CODE_OF_CONDUCT.md`.

## How to contribute
1. Fork the repo.
2. Create a topic branch: `git checkout -b feat/short-description`.
3. Run tests and linting locally.
4. Open a Pull Request with a clear description, test notes, and linked issues.

## Style
- Python: follow PEP8, black formatting recommended.
- JavaScript: follow ESLint (if present).
- Commit message style: `type(scope): short description` (e.g., `feat(cli): add 'dex status' command`).

## PR checklist
- [ ] Tests added or updated
- [ ] Linting passes
- [ ] Documentation updated
